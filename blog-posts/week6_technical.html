Blocks, Procs, and Lambdas

Blocks, Procs, and lambdas are all closures in the Ruby language. Closures are functions which remember all of the values of the variables in the function's scope when it was created. When a closure is created, it keeps the values of the variables that were in the scope when the closure was made or defined. This way, the closure is able to access the variables when it is called even if the variables are no longer in the scope. Furthermore, closures have the ability to be passed around like an object (stored in collections or passed on to other functions).

Lambdas

Lambdas are almost identical to Procs with a few differences. The first is that instead of writing lambda.new like we would write proc.new, we just write 'lambda'. The second is that a lambda checks the arguments that are passed into it, but a proc does not. So, if you pass the wrong number of arguments, the lambda will throw an error while a proc will just assign nil to any arguments that are missing. Finally, once a lambda returns something, it passes the control back to the calling method. A proc does not pass control back to the calling method.
